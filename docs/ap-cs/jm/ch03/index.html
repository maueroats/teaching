Classes (3.3)

  1. What is a class, according to Java Methods?

  2. What is the difference between a class and an object? Give two examples not from the text.

  3. Are there any rules in Java governing the name of a file containing DinosaurBot? If so, what?

  4. Does Java force you to name your classes beginning with capital letters? Explain the context of this question.

  5. What is an instance variable? How does it compare to a field?

  6. What vocabulary word is used for behaviors of a class?

  7. Give an example of an “implicit” field, constructor, or method. (Appears later in the chapter.)

  8. If you refuse to use import statements, does that mean you cannot access information from other files, like the Color class? Explain.

  9. Does Java care what order methods and fields appear in a class?

  10. What conventions do Java programmers use when ordering the things that they put in a class?

  11. Is it considered good design to put the main method inside your WorkerBot class? What do the Java Methods authors say? Why do you think they say that?

Fields, Constructors, Methods (3.5)

  1. Is it an error to have two constructors: public Bug() and public Bug(Color c)? If not, how do you determine which one is used?

  2. What is a “reference” (intuitively)?

  3. What is garbage collection?

  4. Is it possible for a class to have no constructors? How? Or why not?

  5. What does it mean to “send a message” to an object in Java (metaphorically speaking)?

  6. Does a method belong to an object or a class? Explain briefly.

  7. How do you indicate that a method gives no result?

  8. There is a technique mentioned called either encapsulation or information hiding. What is one way to implement encapsulation?

  9. What is one advantage of encapsulation? (One of the three mentioned in the book is more important than the other two - try to pick that one.)
